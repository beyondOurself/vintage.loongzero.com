# PRD

## 产品概述

- 本项目用于产出**古着（Vintage）电商**方向的**高保真 HTML 原型**，外观接近真实移动端 App，支持在 `index.html` 中一览全部界面，并支持后续导入 Figma 编辑。
- **终端形态**：以**移动端**为主；原型阶段聚焦信息架构与交互参考，**后续**可落地为小程序、原生/Hybrid App，本仓库页面命名与模块划分应对齐上述形态，避免与最终产品脱节。

## 目标用户

- 产品与设计评审、前端实现参考。

## 业务与信息架构（原型需覆盖）

1. **商城核心能力**（按迭代逐步补页面，独立 HTML 对应）  
   - 首页（推荐/活动入口）、分类与浏览、商品列表与筛选、商品详情。  
   - 购物车、下单与订单列表/详情（含基础状态展示即可）。  
   - 用户中心（账号入口、订单入口、收藏/足迹等占位）。  
2. **优先级**：移动端首屏与主路径清晰；复杂后台、商家端不在本原型 MVP 强制范围，可按后续版本追加。

## 视觉与内容调性

1. **风格关键词**：**古着**、**美拉德（Maillard）** 色系与质感——暖棕、焦糖、燕麦、深咖、做旧与织物感，避免霓虹、廉价促销风与泛白「模板电商」观感。  
2. **素材**：界面所需图片从 Unsplash、Pexels 等来源获取并引用；选图需与上述调性一致（复古、胶片感、大地色、单品特写），忌与古着气质冲突的图。  
3. **设计目标**：参考业内**格调较高**的同类应用（如海外二手/古着/设计师寄售类 App 的信息层级与氛围，**不抄袭**其品牌识别与独占素材），整体要求**有个性、有辨识度**，版式与字体层级统一。  
4. **实现落点**：以 Tailwind 主题变量或 `common.css` 集中定义色板与圆角等，便于全站一致与后续换肤。

## 功能需求（工程与交付）

1. **技术实现**：全部原型界面使用 HTML + Tailwind CSS；可使用 FontAwesome 增强图标与装饰。  
2. **文件组织**：按功能拆分，每个功能界面独立 HTML 文件，命名清晰（示例：`home.html`、`category.html`、`product-detail.html`、`cart.html`、`orders.html`、`me.html`）。  
3. **样式策略**：以 Tailwind 为主；非 Tailwind 覆盖的自定义样式写入独立 CSS 文件并在对应 HTML 中引入。  
4. **主入口**：`index.html` 作为唯一主入口，不在此文件内内联所有界面完整代码。  
5. **集成方式**：通过 **iframe** 加载各子页面；在 index 上**平铺展示**所有子页面（便于一屏浏览）。  
6. **设备外观**：原型区域尽可能模拟 **iPhone 15** 外观（边框、圆角、状态栏区域等）。  
7. **可迁移性**：结构与标注方式需有利于后续**导入/对齐 Figma** 二次编辑。

## MVP 范围

- 建立 `index.html` + iframe 平铺 + iPhone 15 外框样式约定。  
- **古着电商语境下**至少完成一组示例功能页（如 `home` / 商品列表或分类 / `me`）验证文件拆分、主题色与样式复用。  
- 明确一页「设计说明」或根样式中的色板注释（可与 TECH_DESIGN 交叉引用，避免重复堆砌）。

## 后续版本

- 按业务补全购物车、订单流、详情页等独立 HTML 与交互状态。  
- 强化主题变量与暗色/浅色若产品需要。  
- 与小程序、App 团队对齐路由与模块命名（本仓库可为对照表或附录）。

## 非功能需求

- **TOKEN**：在保证可读前提下尽量压缩冗余 HTML/CSS/文案，降低单页与总仓库 TOKEN。  
- **性能**：图片使用合适尺寸与懒加载（若需）。  
- **可用性**：index 平铺时各 iframe 尺寸一致、滚动与预览清晰。
