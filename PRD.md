# PRD

## 产品概述
- 本项目用于产出**高保真 HTML 原型**，外观接近真实移动端 App，支持在 `index.html` 中一览全部界面，并支持后续导入 Figma 编辑。

## 目标用户
- 产品与设计评审、前端实现参考。

## 功能需求
1. **技术实现**：全部原型界面使用 HTML + Tailwind CSS；可使用 FontAwesome 增强图标与装饰。
2. **文件组织**：按功能拆分，每个功能界面独立 HTML 文件，命名清晰（示例：`home.html`、`games.html`、`me.html`）。
3. **样式策略**：以 Tailwind 为主；非 Tailwind 覆盖的自定义样式写入独立 CSS 文件并在对应 HTML 中引入。
4. **主入口**：`index.html` 作为唯一主入口，不在此文件内内联所有界面完整代码。
5. **集成方式**：通过 **iframe** 加载各子页面；在 index 上**平铺展示**所有子页面（便于一屏浏览）。
6. **设备外观**：原型区域尽可能模拟 **iPhone 15** 外观（边框、圆角、状态栏区域等）。
7. **图片**：界面所需图片从 Unsplash、Pexels 等来源获取并引用。
8. **可迁移性**：结构与标注方式需有利于后续**导入/对齐 Figma** 二次编辑。

## MVP 范围
- 建立 `index.html` + iframe 平铺 + iPhone 15 外框样式约定。
- 至少完成一组示例功能页（如 home / 列表 / 个人）验证文件拆分与样式复用。

## 后续版本
- 按实际产品功能补全各独立 HTML 页面与主题变量。

## 非功能需求
- **TOKEN**：在保证可读前提下尽量压缩冗余 HTML/CSS/文案，降低单页与总仓库 TOKEN。
- **性能**：图片使用合适尺寸与懒加载（若需）。
- **可用性**：index 平铺时各 iframe 尺寸一致、滚动与预览清晰。
